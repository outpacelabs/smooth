<h1 align="center">@outpacelabs/squircle</h1>

<p align="center">
  Figma-style corner smoothing (Apple squircles) that keeps your corner radius.
</p>

---

CSS `border-radius` draws a plain circular arc. Figma/Apple corner smoothing
keeps a circular arc of the **same radius** at the corner apex and splices a
bézier ease between it and the straight edges — the corner radius reads
identically, only the transition softens.

Most "squircle" snippets get this wrong: a naive superellipse mapped into the
corner box hugs the corner *tighter* than the circle, so your 20px corners
suddenly read as ~12px. Here the arc keeps your radius, its sweep shrinking
to 90°·(1−s), and the transition is an original construction: one cubic per
side, solved in closed form so curvature is exactly zero where it meets the
straight edge and exactly 1/r where it meets the arc. Both junctions are
G2-continuous, which is smoother than what border-radius itself gives you.
The geometric spec is enforced by property tests (`test/properties.mjs`):
apex on the radius circle, tangency, curvature match, budget clamping.

```bash
pnpm add @outpacelabs/squircle
```

Zero dependencies. `react >= 18` is an optional peer, only needed for the
`/react` entry.

## Path generator (framework-agnostic)

```ts
import { squirclePath } from "@outpacelabs/squircle";

// SVG path data in border-box pixels. smoothing 0–1; 0.6 = Apple/Figma.
const d = squirclePath({ width: 320, height: 200, radius: 20, smoothing: 0.6 });

element.style.clipPath = `path("${d}")`;   // or <path d={d} />, or new Path2D(d)
```

- Radius is clamped to half the shorter side.
- When space is tight, the radius keeps priority and the remaining budget
  goes to smoothing (Figma's preserve-smoothing behavior).
- `smoothing: 0` reproduces plain circular corners exactly.

## React hook

```tsx
import { useSmoothCorners } from "@outpacelabs/squircle/react";

function Card() {
  // radius in px, smoothing 0–100 (default 60, the Apple/Figma target)
  const ref = useSmoothCorners<HTMLDivElement>(20);
  return <div ref={ref} className="rounded-[20px] bg-white/5 …" />;
}
```

Reactive corners: pass `press` and the corners soften to a second
smoothing level while the pointer is down, springing back on release. The
paths share one command structure, so the browser interpolates the clip
natively on the compositor. Instant under `prefers-reduced-motion`.

```tsx
const ref = useSmoothCorners<HTMLButtonElement>(16, 60, { press: 100 });
```

The hook clips with `clip-path: path(...)` and regenerates on resize
(ResizeObserver border-box size, so scale/translate entrance animations never
skew the path). Keep the element's `border-radius` in CSS: it is the no-JS /
unsupported-browser fallback, and the hook zeroes it inline once the clip
applies (letting both paint would clip the smoothed transition back to the
plain circle).

## Caveats

- Smoothing only makes sense on rounded **rectangles**. Capsules
  (`rounded-full` pills) and circles have no straight edge to ease into —
  leave them alone.
- Below ~8px radius the circular-vs-smoothed deviation is sub-pixel.
- `clip-path` clips everything the element paints, including
  `outline`/`box-shadow` of the element itself — draw focus rings on a
  child, or accept slight corner clipping.
